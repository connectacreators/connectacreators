Content pipeline — what's manual, what's automatic

Connecta+ video lifecycle, from script to posted. Only the two green steps are new automation. The purple "Strategist" steps are run by a Content Strategist per client — and until one is hired, they fall back to you (admin).

Manual (you do it) Automatic (system does it) Videographer Editor · Axel Strategist → admin (you, for now) Client

1 · Script row created

Manual

A script line is added to the queue.

You

2 · Script approved

Manual

The script is reviewed and approved to film.

You

3 · Marked filmed

Manual

Marked as filmed — does not show on the editing queue yet.

You

4 · Raw footage uploaded

Manual

Videographer uploads the raw footage (the footage tab).

Videographer

5 · Hand off to the editor

⚡ Automatic

The moment raw footage lands, the system sets the row to Needs Revisions and assigns the client's editor (Axel) — so he knows to start editing. No manual assigning.

Joshua  →  Axel

6 · Editor uploads the cut

Manual

Axel uploads the edit; status is set so it's ready for your review.

Axel

7 · You review the cut

Manual

If it needs work, you leave revisions and assign Axel. If it's good, you schedule it (next step).

You

8 · Approve & send to calendar → Scheduled

Manual

You approve and schedule the post. This flip to Scheduled is what triggers the automation below.

You

9 · Hand off to the client

⚡ Automatic

The moment it's Scheduled, the system sets the assignee → the client and quietly remembers who the editor was, so it can come back to them later.

Axel  →  Client

10 · Send the public calendar link

Manual

You share the public link so the client can watch and review on their phone.

You

11 · Client reviews on the public link

Two outcomes:

Approves Manual

You post / mark Published.

Requests revision ⚡ Auto

System flips to Needs Revisions, sends it back to Axel, and drops the client's timestamped notes into the editing queue.

↺ Revision loop · Axel edits → you re-check → schedule → back to client → … until approved

12 · Client approved → Post

Manual

Once the client approves, it's posted / Published.

Client  →  You
What's being built (3 green automations + roles + assignability):
① Step 5 — when raw footage is uploaded, auto-set Needs Revisions + assign the client's editor (Axel).
② Step 9 — auto-assign the row to its client when it becomes Scheduled (and remember the editor).
③ Step 11 — when the client requests a revision, auto-set Needs Revisions + reassign the remembered editor + attach the timestamped notes.
④ Plus: a Content Strategist role (per client, admin fallback) and the row's own Connecta+ client selectable in the assignee dropdown.
