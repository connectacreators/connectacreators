Script Editor — three ways to un-crowd it

Same script (Bravo Bonetti · "Los 5 estados…") rendered in each layout. Scroll to compare.

A · Side rail

Recommended

The script becomes the page. Everything that is *about* the script (format, winning idea, caption, footage) moves into a slim right rail — always visible, never in the way. Save lives in a sticky top bar with the secondary actions folded into a ⋯ menu. This matches how you actually work: 90% of time in the document, glancing at setup.

B · Collapsed setup rows

Lowest risk

Same single column, 80% less chrome. The three setup cards compress into one-line rows that expand inline when clicked. The doc starts ~450px higher. Cheapest to build, smallest change to learn — but setup content is hidden until clicked, and expanding pushes the doc down while open.

C · Tabs

Most focus

One thing on screen at a time. Script is the default tab; Setup (format + winning idea + footage) and Caption get their own spacious views. Cleanest possible writing surface — but out of sight is out of mind, and hopping tabs to check the winning idea while writing gets old fast.
